# Clean Code Principles

## Single Responsibility Principle
- Each class, function, or module should have only one reason to change
- Limit functions to performing a single logical operation
- Keep functions under 20-30 lines of code when possible

## DRY (Don't Repeat Yourself)
- Extract duplicate code into reusable functions or classes
- Create shared utilities for common operations
- Use inheritance or composition to share functionality between related components

## KISS (Keep It Simple, Stupid)
- Choose the simplest solution that meets requirements
- Avoid premature optimization and over-engineering
- Prefer readable code to clever tricks

## YAGNI (You Aren't Gonna Need It)
- Don't build features until they're needed
- Avoid speculative generalization
- Focus on current requirements first

## Readable Code
- Write code that is easy to understand at first glance
- Use meaningful variable names that express intent
- Structure code for readability, not just functionality

## Maintainable Code
- Write code that can be easily modified by others
- Follow consistent patterns across the codebase
- Consider future extension points
